mechanism

How self-pruning works

Four steps — trained end-to-end with a single optimizer. No separate pruning phase.

01 · GATE SCORES

One gate per weight

Every weight gets a companion learnable number called a gate score — same shape, updated by the same optimizer.

02 · SIGMOID

Squash to (0, 1)

Sigmoid converts gate scores to values between 0 and 1. A gate of 0 means that weight is silenced — effectively deleted.

03 · MASK

Multiply onto weights

Pruned weight = weight × gate. When gate ≈ 0, the connection contributes nothing regardless of the weight's magnitude.

04 · L1 PRESSURE

Penalise open gates

A sparsity penalty pushes all gates toward zero. Gates only stay open if the accuracy benefit outweighs the penalty λ.

# PrunableLinear forward pass gates = torch.sigmoid(self.gate_scores) # ∈ (0,1) pruned_weights = self.weight * gates # gate=0 kills it output = F.linear(x, pruned_weights, self.bias) # Total loss loss = CrossEntropy(logits, labels) + λ × sum(all gates)

why it works

The intuition

Written for curious non-engineers. No equations required to understand.

Why L1 creates hard zeros

There are two ways to penalise numbers: L1 (sum) and L2 (sum of squares). L1 applies the same pressure no matter how small a value already is — so it can push all the way to exactly zero. L2 loses pressure near zero and never fully closes a gate.

L2 gradient → 0 as value → 0 L1 gradient = constant always

Survival of the fittest connections

Every gate faces constant survival pressure (λ). To survive, a connection must prove its worth by improving predictions. Redundant connections — where two paths carry the same signal — lose the competition and get pruned.

Why warm-up matters

Applying full pruning pressure from epoch 1 kills the network before it learns anything useful. The warm-up lets the network first discover what's important, then removes what isn't. Like pruning a tree — only after it's grown.

ep 0–16: λ=0 free learning ep 16–36: λ ramps gentle pressure ep 36–80: λ=full pruning

Better than post-training pruning

Traditional pruning removes weights after training is done — surviving weights never adapt. Self-pruning lets weights co-adapt in real time, filling in for their pruned neighbours. The result is a stronger sparse network.
